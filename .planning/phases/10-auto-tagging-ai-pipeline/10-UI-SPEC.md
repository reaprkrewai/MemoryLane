## Color

| Role | Light mode value | Dark mode value | Usage |
|------|-----------------|-----------------|-------|
| Dominant (60%) | `#FFFFFF` (`--color-bg`) | `#0A0A0D` (`--color-bg`) | Page background, TagRow background |
| Secondary (30%) | `#F9F8F6` (`--color-surface`) | `#141418` (`--color-surface`) | Ghost chip hover bg (`hover:bg-surface`) |
| Accent (10%) | `#D97706` (light) / `#8B7FFF` (dark) (`--color-accent`) | same | Sparkle hover/focus icon, ghost chip hover border, focus rings |
| Destructive | `#DC2626` (light) / `#FF7A7A` (dark) | same | Not used in this phase |

**Accent reserved for (this phase):**
- Sparkle button icon on hover: `hover:text-accent`
- Sparkle button focus ring: `focus-visible:ring-accent`
- Ghost chip border on hover: `hover:border-accent/40`
- Settings toggle selected state (OptionButton pattern): `bg-accent text-amber-950 border-accent`

**Ghost chip color contract (locked from CONTEXT.md D-12):**

| State | Background | Border | Text | Border style |
|-------|-----------|--------|------|-------------|
| Default | `bg-transparent` | `border-muted/50` (`--color-text-muted` at 50% opacity) | `text-muted` | `border-2 border-dashed` |
| Hover | `hover:bg-surface` | `hover:border-accent/40` | `hover:text-text` | dashed (unchanged) |
| Focus-visible | `bg-surface` | `border-accent/40` | `text-text` | dashed (unchanged) |

The dashed border color is `--color-text-muted` at 50% opacity — NOT `--color-border`. This is intentional: `border-border` (`--color-border`) is used by real UI chrome, so ghost chips use muted/50 to signal their "pending, not committed" status without competing with structural borders.

**New-tag prefix icon:** `<Plus size={10} className="mr-1 -ml-0.5 inline flex-shrink-0" />` — only on chips where `isNew === true`. Color inherits from chip text (`text-muted` → `hover:text-text`). (CONTEXT.md D-12)

**Chip body (accept target):**
- `role="button"` + `tabIndex={0}`
- `aria-label="Accept tag suggestion: {name}"`
- Click / Enter / Space → `handleAccept(suggestion)`
- Cursor: `cursor-pointer`

**Dismiss button (X):**
- `<X size={11} />` inside `<button type="button">`
- Size: `h-4 w-4 rounded-full`
- Background on hover: `hover:bg-muted/20`
- Margin: `ml-1`
- `aria-label="Dismiss suggestion: {name}"`
- Click → `handleDismiss(suggestion)`, `e.stopPropagation()`
- Keyboard: Escape key on focused chip → dismiss (handled via `onKeyDown` on chip wrapper)

**Chip render order in TagRow flex-row:**
```
[TagPill...] [TagInput] [SparkleButton] [GhostChip1] [GhostChip2] [GhostChip3]
```
Wraps naturally via parent `flex flex-wrap gap-2`.

### Empty State (inline, transient)

When LLM returns 0 valid suggestions after post-processing:

```html
<span class="text-muted text-xs ml-1 transition-opacity duration-slow">
  No tag suggestions for this entry
</span>
```

- Appears for **4 seconds** then fades to `opacity-0` via CSS transition (start at `opacity-1`, after timeout set `opacity-0`)
- Implementation: `useEffect` with `setTimeout` returning cleanup — set `showEmptyMsg` to true on result, set to false after 4000ms
- Reduced-motion: timeout still fires but transition-duration is overridden to 0.01ms by global reduced-motion guard (instant disappear)
- Do NOT use a toast for this state — it is a content result, not a system notification

**Error state (silent):** No visible UI element. Log to `console.error('[tag-suggestions] ...')`. Sparkle button returns to resting state. (CONTEXT.md D-10)

### Settings Toggle (Settings → AI Features → "Tag suggestions")

**Placement:** Inside `AIFeaturesSection()` in `SettingsView.tsx`. Inserted as a new `<SettingRow>` AFTER the existing "Actions" row (Check Status / Setup Guide buttons) and BEFORE the section's closing tag. Preceded by `<div className="border-t border-border/50" />` divider.

**Widget pattern:** `OptionButton` pair matching the existing "AI Backend" row:
```tsx
<SettingRow
  label="Tag suggestions"
  description="Show a sparkle button in the editor to suggest tags via local AI"
>
  <div className="flex gap-2">
    <OptionButton label="On"  selected={tagSuggestionsEnabled}  onClick={() => handleTagSuggestionsToggle(true)}  />
    <OptionButton label="Off" selected={!tagSuggestionsEnabled} onClick={() => handleTagSuggestionsToggle(false)} />
  </div>
</SettingRow>
```

- Default: "Off" selected (AUTOTAG-06 — locked)
- Selected state: `bg-accent text-amber-950 border-accent shadow-sm` (existing OptionButton pattern)
- Unselected state: `bg-transparent text-text-secondary border-border hover:border-accent/40 hover:text-text hover:bg-surface-secondary`
- ARIA: The OptionButton `<button>` already has implicit role; add `aria-pressed={selected}` to each (researcher discretion on `role="switch"` — OptionButton pair communicates state clearly via visual selected state; `aria-pressed` on each is sufficient)

## Animation & Motion

**Ghost chip appearance:** Chips render with `animate-slide-up` (from animations.css / tailwind.config.js — already defined). Duration: `var(--motion-med)` (300ms). Easing: `var(--ease-out-smooth)`. Each chip gets `style={{ animationDelay: `${index * 60}ms` }}` for a subtle stagger (0ms, 60ms, 120ms for up to 3 chips).

**Reduced-motion behavior:** The global `@media (prefers-reduced-motion: reduce)` stanza in globals.css sets `animation-duration: 0.01ms !important` — chips appear instantly. No additional code needed. The 4-second empty-state timeout still fires; the opacity transition collapses to instant. (CONTEXT.md D-09 / FOUND-04)

**Sparkle button active state:** `button:active { transform: scale(0.98) }` is already in globals.css for all buttons — no additional code needed.

**Loading spinner:** `<Loader2 className="animate-spin" />` uses Tailwind's built-in `animate-spin` (CSS `animation: spin 1s linear infinite`). Reduced-motion: collapses to 0.01ms via global guard — spinner appears frozen; acceptable since the button is `disabled` and the user is waiting.

**Slow-call toast (>30 seconds):** `toast.info("Tag suggestions are taking longer than expected")` via sonner. Info severity. Non-blocking, auto-dismiss. No cancel affordance. (CONTEXT.md D-09, researcher discretion: `info` over `warning`)

## Interaction State Machine (TagSuggestButton)

```
HIDDEN (showSparkle === false)
  → component unmounted, nothing rendered

IDLE (showSparkle === true, suggestions === [])
  → <Sparkles> button, no chips
  → click → LOADING

LOADING (isLoading === true)
  → <Loader2 spin> button (disabled), no chips
  → LLM resolves with ≥1 result → SHOWING_SUGGESTIONS
  → LLM resolves with 0 results → EMPTY_STATE
  → LLM throws → IDLE (silent error, console.error)
  → >30s elapsed → sonner info toast (stays in LOADING)

SHOWING_SUGGESTIONS (suggestions.length > 0)
  → <Sparkles> button (enabled), ghost chips rendered
  → click sparkle → LOADING (replaces chip batch)
  → accept chip → chip removed from state; if last chip → IDLE
  → dismiss chip → chip removed from state; if last chip → IDLE

EMPTY_STATE (suggestions === [], showEmptyMsg === true)
  → <Sparkles> button (enabled), inline message visible
  → after 4s timeout → IDLE (message fades)
  → click sparkle before timeout → LOADING (message cleared)
```
