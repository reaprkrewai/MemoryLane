## Spacing Scale

8-point scale — Chronicle project uses Tailwind spacing units. Declared values:

| Token | px Value | Tailwind class | Usage |
|-------|----------|----------------|-------|
| xs | 4px | `gap-1`, `p-1` | Icon-to-label gaps, tight inline spacing |
| sm | 8px | `gap-2`, `p-2` | Compact element spacing, icon padding |
| md | 16px | `gap-4`, `p-4` | Default intra-widget spacing |
| lg | 24px | `gap-6`, `p-6` | Widget internal padding (matches existing `MoodOverview`, `StatCard` `p-5`/`p-6`) |
| xl | 32px | `gap-8` | Gap between dashboard sections |
| 2xl | 40px | `py-10`, `px-10` | Page-level outer padding (matches existing `OverviewView` `px-10 py-10`) |
| 3xl | 64px | `max-w-[1200px]` | Layout container max-width (matches existing OverviewView) |

**Exceptions:**
- Stat card icon tile: `h-11 w-11` (44px) — meets touch-target minimum for keyboard focus
- Widget cards: `rounded-2xl` border radius — matches existing `StatCard`, `MoodOverview` visual grammar
- Section gap between stat row and main split: `mb-8` (32px) — matches existing layout
- Header margin: `mb-10` (40px) — matches existing OverviewView header

**FAB padding inherited unchanged from v1.0 QuickWriteFAB component; not part of Phase 8 spacing contract.** The FAB occupies a 56px total height as a layout constant (measurement), derived from the existing v1.0 pill geometry — Phase 8 does not introduce or change this value.

## Typography

Four roles, two fonts, two weights. No new type styles introduced in Phase 8 — all roles already defined in `globals.css` and `tailwind.config.js`.

| Role | Size | Font | Weight | Line Height | CSS class / Tailwind |
|------|------|------|--------|-------------|----------------------|
| Label | 11–12px | Inter | 400 (regular) | 1.4 | `text-[11px] text-small-caps` or `text-label` |
| Body | 14px | Inter | 400 (regular) | 1.5 | `text-sm` |
| Heading | 18–20px | Fraunces | 600 (semibold) | 1.2 | `font-display text-lg font-semibold` |
| Display | 44–52px | Fraunces / Fraunces italic | 400 (regular) | 1.0–1.05 | Stat numerals: `font-display font-tabular text-[44px] leading-none`; Greeting: `font-display-italic text-[52px] leading-[1.05]` |

**Rules:**
- Widget titles (`MoodTrends`, `WritingPrompts`, `AIInsights`, `OnThisDay`) use `font-display text-lg font-semibold` — matches `MoodOverview` exactly.
- Timestamp/relative-time labels use `font-display-italic text-xs text-[var(--color-text-muted)]` — matches existing entry rows in OverviewView.
- Stat card labels use `text-small-caps text-[11px] text-[var(--color-text-muted)]` — matches existing `StatCard`. Small-caps visually differentiates Label from Body without needing a separate weight.
- Prompt text in `WritingPrompts` widget uses `text-sm leading-relaxed text-[var(--color-text-secondary)]` — body weight, no emphasis.
- AI Insights summary body uses `text-sm leading-relaxed text-[var(--color-text)]` — readable prose.
- Freshness timestamp (`Generated N minutes ago`) uses `text-xs text-[var(--color-text-muted)]` — secondary metadata.

### 3. MoodTrends (DASH-04, D-08)

**New component:** `src/components/dashboard/MoodTrends.tsx`

**Visual treatment decision (discretion — CONTEXT.md left this open):** Vertical bar chart as inline SVG. Reason: bar chart cleanly shows per-day distribution without ambiguity; lines imply continuity that doesn't exist for moods. Each day = one stacked-bar column. 30 columns (days) across the x-axis.

**Dimensions:**
- Container: `w-full` inside widget card; SVG computed to fill container width
- Bar height range: 40–80px tall SVG viewport; bars stack vertically from bottom
- Column width: auto-computed from `(containerWidth - padding) / 30`
- X-axis: 30 columns, left to right = oldest to newest (today is rightmost)
- Minimal x-axis labels: show only `30d`, `15d`, `today` at three anchor positions, `text-[10px] text-[var(--color-text-muted)]`

**Color:** Each mood segment uses its `MoodOverview` color (listed above) at `opacity-80` for non-zero days; missing-mood days render a `--color-border-subtle` background column at 20% opacity.

**Widget card structure:**
```
rounded-2xl border border-[var(--color-border)] bg-[var(--color-surface)] p-5
  ├── header row: "Mood trends" (font-display text-lg font-semibold) + "last 30 days" (small-caps 10px muted)
  ├── SVG bar chart (mood stacked bars)
  └── mood legend: 5 inline dots with mood-key labels (font-display-italic text-[10px])
```

**Empty state:** When no entries in the 30-day window, render the bar chart area as a row of 30 uniform short columns in `--color-border-subtle` at 30% opacity, with centered text: `"Your mood story unfolds here"` — matches `MoodOverview` empty copy convention.

**Interaction:** Non-interactive for Phase 8 (CONTEXT.md deferred click-to-filter). No hover tooltips required.

**Reduced-motion:** The SVG is static markup; no CSS animations on bars in Phase 8.

### 7. Writing Prompts Widget (DASH-10, DASH-11)

**New component:** `src/components/dashboard/WritingPrompts.tsx`

**Widget card structure:**
```
rounded-2xl border border-[var(--color-border)] bg-[var(--color-surface)] p-5
  ├── header row: "Today's prompt" (font-display text-lg font-semibold) + lightbulb icon (Lucide Lightbulb, 16px, text-muted)
  ├── prompt text block (14px Inter, leading-relaxed 1.6, text-[var(--color-text-secondary)], italic, py-3)
```

**Prompt index logic:**
- Today's index: `dayOfYear(new Date()) % PROMPTS.length` using `getDayOfYear` from `date-fns`
- Offset from `useState<number>(0)`; displayed index = `(todayIndex + offset) % PROMPTS.length`

- Offset resets on unmount (default React behavior for `useState`)

**"Another prompt" element:** Text button, not icon button. Label: `"Another prompt"` with `ArrowRight` icon (12px) inline — `group flex items-center gap-1 text-xs`. This is a discretion decision (CONTEXT.md left it open); text is clearer than an icon for first-time users.

**Empty state:** Not applicable — library always has prompts. Component never renders in empty state (60-item library guarantees a valid index).

## Loading Skeleton Specifications

**Triple-gate rule (CONTEXT.md D-10):** Only show skeletons when ALL THREE conditions are true:
1. `isLoadingPage === true`
2. `allEntries.length === 0`
3. Relevant store primitive is still at its zero/empty initial value

If the user has entries and the store is just refreshing, show stale values — not skeletons.

**Skeleton visual treatment:**
- Background: `bg-[var(--color-surface-secondary)] animate-pulse rounded`
- `animate-pulse` is the Tailwind built-in — uses `opacity` keyframe, already reduced-motion safe in globals.css (global `animation-duration: 0.01ms !important` collapses it to instant)

**Per-widget skeleton shapes:**

| Widget | Skeleton shape |

## Interaction Specifications

### Global Keyboard Shortcut — `Ctrl/Cmd+N`

- Handler location: `src/hooks/useGlobalShortcuts.ts` (new file, shape from `useIdleTimeout.ts`)
- Mounted in `App.tsx` inside the unlocked state branch — never fires when PIN lock is active
- Condition: `(e.ctrlKey || e.metaKey) && e.key === "n" && !isTypingContext()`
- `isTypingContext()`: returns true when `document.activeElement` is `INPUT`, `TEXTAREA`, or has `contenteditable="true"` — prevents hijacking TipTap or any text input
- On fire: `createEntry()` → `selectEntry(newId)` → `navigateToEditor(activeView)` — reuses existing flush-and-navigate pattern
- `window.addEventListener("keydown", handler)` with capture=false; cleanup in `useEffect` return

### FAB Click / Shortcut Behavior When Editor Is Open

- CONTEXT.md D-24: creates new entry, switches editor to it. `selectEntry` already calls `flushAndClearTimers()` — no work is lost.
- No confirmation dialog needed (auto-save is always active).

### AI Insights Refresh Flow

1. User clicks `"Refresh insight"` button
2. Check `aiStore.available`: if false → silent no-op (no toast)
3. Check last-7-days entry count: if 0 → show "Not enough entries yet" toast or inline message
4. Show `isGenerating` skeleton state (spinner on Refresh icon, body becomes skeleton bars)
5. Call `insightService.generateWeeklySummary()` → `hybridAIService.askQuestion(prompt, context)`
6. On success: write `ai_insight_text` + `ai_insight_generated_at` to `settings` KV table via `INSERT OR REPLACE`; update local component state
7. On failure: silently log error; re-show empty/unavailable state; no toast, no dialog

### OnThisDay Expand/Collapse

Existing behavior: toggle via header button click, state in `uiStore.onThisDayCollapsed`. No changes for Phase 8.

### WritingPrompts Cycling

"Another prompt" click increments local `offset` state. No persistence, no animation between prompts (Phase 11 concern). Prompt swaps instantly.

### Error States

**Phase 8 design principle:** No error dialogs or toast errors from dashboard widgets. All error conditions silently degrade to the empty/unavailable state above. This matches v1.0's `hybridAIService` graceful fallback convention.

| Scenario | UX |
|----------|----|
| `getEntryStats()` DB failure | Propagates to App-level error boundary (no per-widget handling per CONTEXT.md D-11) |
| AI Insights LLM call fails | Silent — re-show "No summary yet" state, log to console in dev |
| AI Insights: Ollama down on Refresh insight click | Silent no-op |

## Design Decisions Log (Discretion Items Resolved)

Per 08-CONTEXT.md `### Claude's Discretion` — all discretion items resolved here:

| Item | Decision | Rationale |
|------|----------|-----------|
| `MoodTrends` chart type | Vertical stacked bar chart as inline SVG | Bars cleanly show daily distribution; lines imply false continuity for categorical mood data |
| Streak `/7 this week` micro-copy | `value={min(streak,7)}` + `suffix="/7"` + `label="this week"` | Compact; existing `suffix` prop handles the rendering; label provides the word "week" |
| `WritingPrompts` "Another prompt" | Text button with inline `ArrowRight` icon | Text is clearer than icon-only for first-time users; matches "View all" link style |
| AI Insights auto-run on first open | No — wait for explicit Refresh insight click | LLM calls can take 10–30s; respects battery and user agency; avoids surprise on landing |
| FAB z-index behind modals | Keep `z-40` (existing) — modals/popovers use Radix `z-50`+ | Current z-40 naturally stacks behind modal overlays |
| Pulse skeleton threshold | Triple-gate: `isLoadingPage && allEntries.length === 0 && primitive === 0/[]` | Skeletons = "loading", not "empty" — distinguishes the two states |
| InsightService layout | Single `generateWeeklySummary()` export + private helper functions | Matches `aiSettingsService.ts` pattern; one clear entry point |
