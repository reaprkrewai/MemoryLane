### 2.1: Database Schema Extension

**File:** `src/lib/db.ts`

Add `media_attachments` table to track photo metadata:

```sql
CREATE TABLE IF NOT EXISTS media_attachments (
  id TEXT PRIMARY KEY,
  entry_id TEXT NOT NULL,
  photo_path TEXT NOT NULL, -- Local filesystem path or relative path
  thumbnail_path TEXT NOT NULL, -- Path to 100x100px thumbnail
  file_size INTEGER,
  mime_type TEXT DEFAULT 'image/jpeg',
  display_order INTEGER DEFAULT 0,
  created_at INTEGER NOT NULL,
  FOREIGN KEY (entry_id) REFERENCES entries(id) ON DELETE CASCADE
);

CREATE INDEX idx_media_attachments_entry ON media_attachments(entry_id);
CREATE INDEX idx_media_attachments_order ON media_attachments(entry_id, display_order);
```

### 2.7: Fetch Photos with Entry Query

**File:** `src/stores/entryStore.ts`, `src/lib/db.ts` (extend)

When fetching entries, also fetch associated photos:

```typescript
// Update getAllEntries() to include photos
const getAllEntries = async (): Promise<Entry[]> => {
  const entries = await db.select<Entry[]>(`
    SELECT e.*, COUNT(m.id) as photoCount
    FROM entries e
    LEFT JOIN media_attachments m ON e.id = m.entry_id
    GROUP BY e.id
    ORDER BY e.date DESC
  `);

  // Fetch photos for each entry in parallel
  const entriesWithPhotos = await Promise.all(
    entries.map(async (entry) => ({
      ...entry,
      photos: await getPhotosForEntry(entry.id),
    }))
  );

  return entriesWithPhotos;
};

// New helper
const getPhotosForEntry = async (entryId: string): Promise<Photo[]> => {
  return await db.select<Photo[]>(
    'SELECT * FROM media_attachments WHERE entry_id = ? ORDER BY display_order ASC',
    [entryId]
  );
};
```

**Acceptance Criteria:**
- [ ] Entry query includes photos via JOIN
- [ ] Photos are fetched in display order
- [ ] Large photo lists don't slow down entry fetch
- [ ] Photos are hydrated when entry is selected

## Implementation Notes

- **File Storage:** Use Tauri's `@tauri-apps/plugin-fs` to read/write files. App data directory is `appDataDir()` from Tauri config.
- **Thumbnail Generation:** Use `sharp` (Node.js) or `canvas` (browser). For Tauri, use Rust native image lib via Tauri plugin. Alternatively, use browser Canvas for client-side thumbnail creation if performance is acceptable.
- **Photo Limits:** 5-10 photos per entry. Enforce on UI (disable button after limit reached) and DB (validation in insert).
- **File Size:** No explicit limit in v1. Consider reasonable limit (e.g., 10MB per photo) if needed.
- **Drag-to-Reorder:** Deferred to v2. Current schema supports it via `display_order` column, but UI interaction is not needed for v1.
