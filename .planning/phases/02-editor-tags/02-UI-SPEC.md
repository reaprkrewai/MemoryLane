## Spacing Scale

Declared values (multiples of 4 only):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, tag pill internal padding, bubble menu item gaps |
| sm | 8px | Sidebar entry list item padding, mood button padding, tag row gap |
| md | 16px | Editor horizontal padding, metadata bar vertical padding, section margins |
| lg | 24px | Editor content top/bottom padding, sidebar section gaps |
| xl | 32px | Editor max-width gutter, major layout breaks |
| 2xl | 48px | Page-level vertical breathing room |
| 3xl | 64px | Not used in Phase 2 |

Exceptions:
- Touch/click targets for mood buttons: minimum 36x36px (satisfying interaction requires adequate target)
- Bubble menu: 6px gap between buttons (between xs and sm, acceptable for toolbar density)
- Tag pills: 6px vertical / 10px horizontal internal padding (visually balanced pill shape)

## Typography

All sizes declared in tailwind.config.js `fontSize` extension — use these utility classes directly.

| Role | Class | Size | Weight | Line Height | Usage |
|------|-------|------|--------|-------------|-------|
| Body | `text-body` | 14px | 400 | 1.5 | Editor prose, sidebar entry metadata, tag autocomplete items |
| Label | `text-label` | 12px | 400 | 1.4 | Word count, char count, mood label on hover, tag usage count in dropdown, date display |
| Heading | `text-heading` | 20px | 600 | 1.2 | Not used directly in Phase 2 editor; reserved for section headers |
| Display | `text-display` | 28px | 600 | 1.15 | Not used in Phase 2 |

Editor prose typography:
- TipTap editor body text: 14px / weight 400 / line-height 1.6 (slightly looser than UI body for readability)
- H1 in editor: 20px / weight 600 / line-height 1.2 (maps to `text-heading`)
- H2 in editor: 16px / weight 600 / line-height 1.3
- Blockquote in editor: 14px / weight 400 / italic / left border 3px `var(--color-accent)` / color `var(--color-text-muted)`
- Code inline: 13px / monospace / `bg-surface` background / 4px horizontal padding / 2px vertical padding

Maximum 2 font weights in Phase 2 UI chrome: 400 (regular) and 600 (semibold).

## Color

Source: `src/styles/globals.css` — Chronicle custom tokens. Use Tailwind aliases (`bg-bg`, `bg-surface`, `text-accent`, etc.).

| Role | Light Value | Dark Value | Tailwind alias | Usage |
|------|-------------|------------|----------------|-------|
| Dominant (60%) | `#FAFAF9` | `#111110` | `bg-bg` | App background, editor writing surface |
| Secondary (30%) | `#F0EFED` | `#1C1B1A` | `bg-surface` | Sidebar background, metadata bar background, tag dropdown, bubble menu |
| Accent (10%) | `#F59E0B` | `#F59E0B` | `text-accent` / `bg-accent` | Reserved — see list below |
| Destructive | `#DC2626` | `#EF4444` | `text-destructive` | Delete entry confirmation button only |

**Accent (`#F59E0B`) is reserved for exactly these elements:**
1. "New Entry" button — background fill
2. Selected mood button — icon color highlight
3. Active/selected tag pill — border or background tint
4. Auto-save "Saved" indicator text (brief flash only, not persistent)
5. Bubble menu active formatting button (e.g., Bold when bold is active)

**Accent is NOT used for:** hover states, borders, focus rings (use `--color-border` / `--ring` instead), general interactive elements, or the sidebar navigation active state (use `bg-surface` + bold weight instead).

Muted text color `#6B7280` (light) / `#9CA3AF` (dark) — use for word count, char count, date label, tag usage count, placeholder text.
Border color `#E5E3DF` (light) / `#2A2927` (dark) — use for metadata bar bottom border, tag row top border, bubble menu border.

## Component Contracts

### Metadata Header Bar (above editor, D-06)

Layout: horizontal flexbox, `bg-surface`, bottom border `1px solid var(--color-border)`, height 48px, horizontal padding 16px.

- **Left zone (date):** Text rendered as "Wednesday, Apr 9" in `text-label` / `text-muted`. Clicking opens a Popover (shadcn Popover + Calendar) for date + time selection. Cursor pointer. On hover: text color shifts to `text-text`.
- **Center zone (mood selector):** 5 icon buttons in a row, 8px gap between them. Each button is 36x36px. Unselected: icon in `text-muted`, no background. Selected: icon in `text-accent`, `bg-surface` or subtle amber tint background (`bg-amber-50` / dark: `bg-amber-950/20`). Hover: icon shifts to `text-text`. Transition: 150ms ease for color and scale. On selection: brief scale(1.2) pop returning to scale(1) over 150ms.
- **Right zone (counts):** Word count and character count side-by-side, `text-label` / `text-muted`. Format: "142 words · 847 chars". Updates in real time as user types. Right-aligned in the right zone.

Mood icons (lucide-react):
- great → `Smile` (or `Laugh`)
- good → `SmileIcon` variant (use `CircleSmile` if available, else `Smile`)
- okay → `Meh`
- bad → `Frown`
- awful → `CloudRain` or `Angry`

Decision: Use lucide icons, not emoji. Consistent with existing icon library usage (D-02 context note, Claude's discretion on icon choices). Settled: `Laugh`, `Smile`, `Meh`, `Frown`, `Angry` — 5 distinct levels readable at 18px.

### TipTap Editor Surface

Container: `bg-bg`, no border, no box shadow. Full width within the content area. Horizontal padding 32px on left and right (creates comfortable reading measure). Vertical padding 24px top, 32px bottom.

Max content width: 680px centered (optimal reading line length for prose). Below 768px viewport, max-width collapses to full width minus 16px padding each side.

Placeholder text (when editor is empty): "Start writing…" in `text-muted`, `text-body`, not italic.

Cursor: default text cursor. No custom cursor styling.

### Bubble Menu (D-01, D-02)

Appears on text selection. Positioned above the selection by 8px gap.

Container: `bg-surface`, `border border-border`, `rounded-md`, `shadow-md`, `px-1 py-1`, flex row, gap 2px.

Buttons: 7 total — Bold (B), Italic (I), H1, H2, Bullet List, Blockquote, plus Code (code inline added for practical completeness within EDIT-01 scope).

Each button: 28x28px, `rounded-sm`, `text-label` or 13px icon size. Inactive: `text-text` / `hover:bg-bg`. Active (format currently applied): `text-accent` background tint.

Keyboard shortcuts shown in tooltip on hover (delay 500ms): Cmd+B, Cmd+I — standard browser behavior, no custom implementation needed.

### Tag Row (below editor, D-07, D-08, D-09)

Container: sits directly below the editor content area, separated by `1px solid var(--color-border)` top border. Background: `bg-bg`. Padding: 12px 32px (matching editor horizontal rhythm).

Layout: horizontal flex wrap. Existing tag pills on the left, tag input field at the end of the row.

**Tag pills:** Pill shape, `rounded-full`. Background: 15% opacity of the tag's assigned color (tinted background, not just a dot). Border: 1px solid the tag's assigned color at 40% opacity. Text: tag name in `text-label` (12px) / weight 400, color derived from tag color (darkened for contrast). Remove button: `×` appearing on hover at the right side of the pill, 14px, same color as text.

Tag assigned color tinting pattern: use CSS `color-mix(in srgb, {tag-color} 15%, transparent)` for bg, `color-mix(in srgb, {tag-color} 40%, transparent)` for border. This produces a readable, distinctive pill without overwhelming the editor surface.

**8 preset tag colors:**
1. `#EF4444` — Red
2. `#F97316` — Orange
3. `#EAB308` — Yellow
4. `#22C55E` — Green
5. `#3B82F6` — Blue
6. `#8B5CF6` — Purple
7. `#EC4899` — Pink
8. `#6B7280` — Gray

**Tag input field:** Borderless, no background. Placeholder: "Add tag…" in `text-muted`. Width: 120px minimum, grows with typed content. On focus: no border appears (stays borderless to feel inline).

**Autocomplete dropdown:** Appears below the tag input, `bg-surface`, `border border-border`, `rounded-md`, `shadow-md`. Max height 200px, scrollable. Each item: tag color swatch (8x8px circle) + tag name (`text-body`) + usage count (`text-label` / `text-muted`, right-aligned). Active item: `bg-bg` highlight. Press Enter or click to assign. Type new name + Enter to create (dropdown shows "Create tag '{name}'" as last item when no exact match).

**Color picker (post-creation, D-09):** Clicking an existing tag pill opens a small popover with 8 color swatches (20x20px circles, 4px gap). Current color has a checkmark. Clicking a swatch reassigns the color and closes the popover. No cancel needed — last color is preserved until user changes it.

### Sidebar Entry List (D-03, D-04)

Sidebar background: `bg-surface` (already established in Phase 1 AppShell).

**"New Entry" button:** Full width within sidebar, amber fill (`bg-accent` / `text-white` or `text-stone-900` for contrast), `rounded-md`, height 36px, `text-body` / weight 600. Label: "New Entry". Pinned at the top of the Journal section, 8px margin below it before the entry list.

**Entry list items:** Each item is a clickable row, 56px min-height, horizontal padding 12px, vertical padding 8px. Three data points in a single line:
- Date: `text-label` / `text-muted` left-aligned (e.g., "Apr 9")
- Mood icon: 14px lucide icon, `text-muted`, center
- Word count: `text-label` / `text-muted` right-aligned (e.g., "142w")

Active/selected entry: `bg-bg` background (slightly lighter than sidebar surface). Hover: `bg-bg` at 60% opacity. No left border accent — avoid visual noise in the lean list.

List is scrollable. No pagination UI — loads all entries (Phase 2 scope is small data sets; keyset pagination is Phase 3).

Delete entry: accessible via right-click context menu or a trash icon that appears on hover (far right, 14px `Trash2` icon in `text-muted`, shifts to `text-destructive` on hover). Clicking opens a confirmation dialog.

### Auto-Save Indicator (Claude's discretion — D-04)

Silent by default. When save completes: a brief "Saved" text appears in the metadata bar right zone, replacing counts for 1.5 seconds, then fades back to counts. Text: `text-label` / `text-accent`. No toast notification for routine saves.

On save failure only: use `sonner` toast — "Couldn't save entry. Check disk space." — error variant. Sonner is already initialized in App.tsx.

### Delete Entry Confirmation (Claude's discretion)

shadcn `AlertDialog`. Title: "Delete this entry?" Body: "This entry will be permanently deleted. You can't undo this." Cancel button: secondary variant. Confirm button: destructive variant, label "Delete Entry". No keyboard shortcut for deletion — intentional friction.

## Interaction States

| Element | Default | Hover | Active / Selected | Disabled |
|---------|---------|-------|-------------------|----------|
| New Entry button | amber bg | slightly darker amber (`#D97706`) | pressed scale(0.97) | not applicable |
| Mood button | muted icon | text-text icon | amber icon + tint bg | not applicable |
| Tag pill | tinted bg + border | show × remove button | darker border | not applicable |
| Sidebar entry row | surface bg | bg-bg highlight | bg-bg + none border | not applicable |
| Bubble menu button | text-text | bg-bg highlight | amber text + tint | not applicable |
| Tag autocomplete item | transparent | bg-bg highlight | bg-bg highlight | not applicable |
| Delete icon (entry) | hidden | visible muted | visible destructive | not applicable |

Transition duration: 150ms ease for all color/opacity transitions. 150ms ease-out for scale transforms.
Focus rings: use `ring-2 ring-ring ring-offset-2` (shadcn default, maps to amber `--ring` token).
