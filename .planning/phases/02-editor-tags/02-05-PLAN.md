```typescript
useEffect(() => {
  // Read persisted interval on mount to sync the UI selector
  (async () => {
    try {
      const { getDb } = await import("../lib/db");
      const db = await getDb();
      const rows = await db.select<{ value: string }[]>(
        "SELECT value FROM settings WHERE key = 'autosave_interval'"
      );
      if (rows.length > 0) {
        const ms = parseInt(rows[0].value, 10);
        if ([5000, 10000, 30000].includes(ms)) setAutoSaveMs(ms);
      }
    } catch {
      // ignore — default 5000 is correct
    }
  })();
}, []);
```

**Step 4: Add the interval selector to the right zone.**

Modify the right zone `div` to render the interval selector alongside the word/char count. The selector sits to the LEFT of the count (with a separator dot), or can be placed after the count. Place it after the word/char count with a `·` separator:

```tsx
<div className="flex flex-1 items-center justify-end gap-2">
  {showSaved ? (
    <span className="text-label" style={{ fontSize: "12px", lineHeight: "1.4", color: "var(--color-accent)" }}>
      Saved
    </span>
  ) : (

    </span>
  )}
  <span className="text-label text-muted" style={{ fontSize: "12px" }}>&middot;</span>
  <select
    value={autoSaveMs}
    onChange={async (e) => {
      const ms = parseInt(e.target.value, 10);
      setAutoSaveMs(ms);
      await updateAutoSaveInterval(ms);
    }}
    className="cursor-pointer rounded border-none bg-transparent text-muted outline-none"
    style={{ fontSize: "12px", lineHeight: "1.4" }}
    aria-label="Auto-save interval"
    title="Auto-save interval"
  >
    <option value={5000}>Save 5s</option>
    <option value={10000}>Save 10s</option>
    <option value={30000}>Save 30s</option>
  </select>
</div>
```

**Why a native `<select>` instead of a shadcn `Select`:** This is gap closure — minimal change. A native `<select>` requires zero new imports, no popover dependency, and integrates into the existing single-line MetadataBar height (`h-12`) without layout disruption. The `bg-transparent border-none` styling makes it blend into the bar. Shadcn `Select` adds a popover that would overlap the editor — inappropriate for a subtle inline control.

**Why `bg-transparent border-none`:** The MetadataBar has `bg-surface` background. A bordered select box would look out of place in a tight 48px bar. The native select with `bg-transparent` inherits the surface color and renders as a minimal text-like control consistent with the `text-label text-muted` style of the word count.
