# Plan 06-04: Ollama Setup Wizard & Settings Integration

## Objective

Reduce friction for first-time AI feature users by providing clear, non-technical setup guidance. Show Ollama status in settings, enable re-checking availability without restarting the app.

**Purpose:** Make it easy for non-technical users to install Ollama, understand why it's needed (privacy), and enable AI features without frustration.

**Output:**
- Ollama setup wizard modal (explains + guides 3-step setup)
- Settings page AI status section
- "Enable AI" / "Check Again" buttons
- Setup wizard triggered on first AI search attempt if Ollama unavailable

## Context

### Setup Wizard Trigger

From 06-CONTEXT.md, decision 8: "On first semantic search attempt, check if Ollama is running. If not, show guided setup wizard."

**Trigger points:**
1. User clicks "AI Search" tab in SearchView
2. searchStore checks aiStore.available
3. If false → show OllamaSetupWizard modal
4. User follows 3 steps or clicks "Skip for now"

**Skip for now:** Disables wizard for this session, but wizard re-appears on app restart.

### Wizard Content

Three steps:
1. **Download Ollama**
   - Text: "Ollama lets you run AI models privately on your computer. All your journal entries stay on your device."
   - Link: https://ollama.com/download
   - Button: "Download Ollama"

2. **Install & Start Ollama**
   - Text: "Follow the installer. Once installed, Ollama will start automatically."
   - No action required in wizard (user goes to ollama.com)

3. **Pull the embedding model**
   - Text: "Open your terminal and run: `ollama pull nomic-embed-text`"
   - For Q&A, also: `ollama pull llama2:7b`
   - Button: "Copy commands to clipboard" (optional nice-to-have)

**After setup:**
- "Check Again" button re-runs health check
- If Ollama available → close wizard, proceed to search
- If still unavailable → show retry message

## Notes

- **Privacy messaging:** Wizard emphasizes local computation ("stays on your device", "no cloud"). This differentiates Chronicle AI from other journal apps.
- **Copy-to-clipboard:** "Copy commands" button on step 3 is nice-to-have but not essential for v1 (user can manual type command).
- **Model selection:** v1 uses hardcoded llama2:7b. Future version could let user choose 7b vs 13b in settings (deferred to v2).
- **Ollama restart:** If user restarts Ollama after app is running, health check is called periodically (not in this plan, but could be added in future to auto-detect Ollama coming online).
- **Wizard re-appearance:** Currently appears every app restart if skipped. Could be changed to "skip permanently" with a settings toggle (v2 enhancement).
