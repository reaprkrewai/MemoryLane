**Tag Management table row density (locked):**

- Actions column width: `w-24` (96px) — accommodates rename icon + delete icon with `gap-2`
- Sort control: `mb-4` (16px) gap below header before table

**Exceptions:** `py-3` (12px) on Tag Management rows — data-table density; see above.

### Per-Animation Implementation Contracts

**ANIM-01 — Dashboard stagger-in (50ms per card):**
- Mechanism: existing `.stagger-children > *` + `--i` CSS custom property from animations.css
- Parent: `<div className="stagger-children">` wrapping the stat cards grid
- Each card: `style={{ "--i": index } as React.CSSProperties}` (inline prop cast)
- Animation class on each card: `animate-fade-in` (existing token, 200ms `--motion-med`)
- Max stagger: 6 cards × 50ms = 300ms total offset; base fade-in plays on top of delay
- Reduced-motion: global stanza collapses all animation-duration to 0.01ms — cards appear instantly

**ANIM-02 — TagPill pop-in / pop-out:**
- Pop-in: apply `animate-tag-pop-in` on first mount of a TagPill. Since TagPill is always newly mounted when added (React key = tag id), the class fires once automatically.
  - Exact class string: `animate-[tag-pop-in_150ms_ease-out_both]` OR the `animate-tag-pop-in` Tailwind token
- Pop-out: requires deferred removal. Wrap `onRemove` handler: start `animate-tag-pop-out` on the pill wrapper, then call actual `tagStore.removeTagFromEntry` after 120ms timeout (or use `onAnimationEnd` callback).
  - Exact class string applied during removal: `animate-[tag-pop-out_120ms_ease-in_both]`
  - State: `isRemoving: boolean` on each TagPill or controlled from TagRow

**ANIM-03 — Radix Popover / AlertDialog scale + fade:**
- Both Radix components already carry `data-[state=open]:animate-in data-[state=closed]:animate-out data-[state=closed]:fade-out-0 data-[state=open]:fade-in-0 data-[state=closed]:zoom-out-95 data-[state=open]:zoom-in-95` in their existing wrappers — confirmed from `alert-dialog.tsx` and `popover.tsx` source.
- Duration: `duration-200` already set on AlertDialogContent. This is 200ms, close to the ANIM-03 spec target of 150ms. DO NOT change the existing class to avoid breaking other callsites.
- The `zoom-in-95` / `zoom-out-95` Tailwind-animate classes cover the `0.95→1.0` scale requirement without any additional keyframe.
- No code changes needed to alert-dialog.tsx or popover.tsx for ANIM-03 compliance.

**ANIM-04 — MoodSelector spring feedback:**
- Mechanism: add `animate-mood-spring` to each mood button's `onClick` via transient state
- Pattern: `const [springing, setSpringing] = useState(false)` inside MoodSelector; on click: `setSpringing(true)`, after 120ms `setSpringing(false)`. Apply `springing ? "animate-mood-spring" : ""` to the button className.
- Alternative (simpler): use Tailwind `active:scale-90` — but cubic-bezier spring feel requires the keyframe. Use the state-driven approach.
- Exact class string on button: `...(springing ? "animate-[mood-spring_120ms_cubic-bezier(0.34,1.56,0.64,1)_both]" : "")`

**ANIM-05 — View crossfade (150ms):**
- Mechanism: plain `animate-fade-out` on outgoing view + `animate-fade-in` on incoming view, both 150ms
- No new keyframe needed: `fade-out` = opacity 1→0, `fade-in` = opacity 0→1 — add `fade-out` Tailwind animation token:
  ```js
  // tailwind.config.js animation additions
  'fade-out': 'fade-out var(--motion-fast) var(--ease-out-smooth) both',
  // tailwind.config.js keyframes additions
  'fade-out': { from: { opacity: '1' }, to: { opacity: '0' } },
  ```
- Implementation: in the view-switch container (AppShell or JournalView), key the view on `activeView`; use React state to track `outgoing` view; render outgoing for 150ms with `animate-fade-out`, then unmount and render incoming with `animate-fade-in`.
- `--motion-fast` = 150ms (confirmed from globals.css Phase 7 token).

**ANIM-06 — Reduced motion audit:**
- The global stanza in `src/styles/animations.css` / `globals.css` (`*, *::before, *::after { animation-duration: 0.01ms !important; transition-duration: 0.01ms !important; }`) covers ALL new keyframes automatically.
- No per-component `useReducedMotion` hook is needed. Verification: run with OS "Reduce motion" enabled and confirm all stagger/pop/spring/crossfade become instant.

```
  → double-click name OR click Pencil → EDITING

EDITING
  → Enter or blur → COMMITTING
  → Escape → READ (discard)

COMMITTING
  → SQL success → READ (name updated in store)
  → SQL error (duplicate) → EDITING + toast.error (name conflict)
```

### Tag Row (delete lifecycle)

```
IDLE (usage_count > 0)
  → delete button: disabled + tooltip visible on hover
  → (no transition — action blocked)

IDLE (usage_count === 0)
  → click Delete → CONFIRM_OPEN

CONFIRM_OPEN (AlertDialog visible)
  → click Cancel or Escape → IDLE
  → click "Delete tag" → DELETING

DELETING
  → SQL success + loadTags() → row removed from list (no animation needed)
  → SQL error → IDLE + toast.error("Failed to delete tag")
```

### Recolor Popover lifecycle

```
CLOSED
  → click swatch → OPEN

OPEN (Popover visible with ColorGrid)
  → click swatch in grid → updateTagColor() → CLOSED (Popover closes)
  → click outside / Escape → CLOSED (no change)
```
