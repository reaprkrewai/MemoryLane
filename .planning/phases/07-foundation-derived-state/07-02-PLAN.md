    ```javascript
          },
    ```

    Indent uses 4 spaces per level matching surrounding lines. Single-quote string style throughout (matches existing `'var(--color-bg)'` at line 9 — D-19 / PATTERNS critical preservation point).

    **Why mirror the keyframes here even though animations.css has them:** Tailwind's JIT generates the `animate-fade-in` class from the `animation` extension, which requires both a `keyframes` entry and an `animation` entry in the config. Without the keyframes mirror, `animate-fade-in` would output `animation: fade-in ...` referencing a name Tailwind doesn't know — the JIT would emit no `@keyframes` declaration. With both definitions present, Tailwind emits its own keyframes (consumed by Tailwind utilities) AND `animations.css` keyframes (consumed by raw CSS like `style={{ animation: 'fade-in var(--motion-med) ...' }}`). The two declarations are byte-for-byte equivalent so the browser deduplicates effectively.

    **Why `both` fill mode:** Animation should hold the final state after running (`forwards`) AND not flash the initial state before starting (`backwards`). `both` is the union.
