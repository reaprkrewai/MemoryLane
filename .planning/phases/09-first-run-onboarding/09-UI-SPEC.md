## Spacing Scale

Multiples of 4. Onboarding overlay uses the existing app scale.

| Token | Value | Usage in Phase 9 |
|-------|-------|-----------------|
| xs | 4px | Privacy bullet icon → text gap; step-dot row gap |
| sm | 8px | Step-dot diameter (1.5 = 6px is the dot, 8px is row gap); footer padding-y |
| md | 16px | Bullet-row vertical gap; AlertDialog header→body gap; spotlight padding around FAB |
| lg | 24px | AlertDialog content padding (`p-6` = 24px); footer→buttons gap; Step 2 Popover internal padding |
| xl | 32px | (not used in this phase) |
| 2xl | 48px | (not used in this phase) |
| 3xl | 64px | (not used in this phase) |

Phase-9 specific exceptions (locked here, were Claude's Discretion in CONTEXT.md):

| Exception | Value | Rationale |
|-----------|-------|-----------|
| AlertDialog max-width | `max-w-md` (448px) for Step 1 + Step 3 | Privacy bullets fit in 3 readable lines at this width; tighter than the shadcn default `max-w-lg` keeps focus on the message |
| Spotlight cutout padding around FAB | **12px** on all sides (computed at runtime: `top -= 12, left -= 12, width += 24, height += 24`) | 8px feels cramped against the FAB pill shape; 12px gives breathing room without revealing too much surrounding chrome. Phase 11 polish may revisit. |
| Step 2 Popover offset from FAB | `sideOffset={16}` (Radix prop) | One spacing-md unit of breathing room between the highlighted FAB and the explanatory tooltip |
| Step 2 Popover max-width | `max-w-xs` (320px) | Tooltip-scale, not modal-scale; copy is one heading + one short paragraph |
| Spotlight backdrop opacity | `rgba(0, 0, 0, 0.6)` | Locks the higher of the two CONTEXT D-09 options — 0.6 reads as "this is a tour", 0.5 felt like "loading state" in dark mode where bg is already very dark |

Exceptions outside the 4-multiple rule:

| Element | Value | Rationale |
|---------|-------|-----------|
| Step indicator dot diameter | 6px (`h-1.5 w-1.5`) | D-10 explicit; below the 8px floor but visually correct for a 3-dot indicator next to text-label sized text |
| Privacy bullet check icon size | 14px (Lucide `<Check size={14} />`) | D-13 explicit; matches optical weight of inline body text |
| HelpSection icon size | 16px (Lucide `<HelpCircle size={16} />`) | D-17 explicit; matches `Database`, `Palette`, `Shield` icons in existing SettingsView SectionHeader |

## Typography

Reuses existing Tailwind `fontSize` tokens from `tailwind.config.js:55-60` — exactly 4 sizes, 2 weights. Inter for body/labels; Fraunces serif (`.font-display`) for the Step 1 welcome heading only.

| Role | Class / Size | Weight | Line Height | Used In |
|------|------|--------|-------------|---------|
| Display | `.font-display text-display` (28px Fraunces) | 600 | 1.15 | Step 1 heading "Welcome to Chronicle AI" only |
| Heading | `text-heading` (20px Inter) | 600 | 1.2 | Step 2 + Step 3 headings; HelpSection SectionHeader title |
| Body | `text-body` (14px Inter) | 400 (default) / 500 (`font-medium` for bullet lead-ins) | 1.5 | All paragraph copy, button labels, "Replay" button label |
| Label | `text-label` (12px Inter) | 500 | 1.4 | "Step N of 3" indicator, "Skip tour" link, SettingRow descriptions |

Two weights total: `400 regular` (body prose, button labels, dialog descriptions) and `500/600` (headings, bullet lead-ins, step indicator). No third weight introduced.

Step-specific typography lockup:

- Step 1 heading: `font-display text-display text-text` — the only place Fraunces appears in this phase, anchoring the "first impression" moment
- Step 1 subheading: `text-body text-text-secondary italic` (the tagline italicized for editorial feel; matches PROJECT.md tagline tone)
- Step 1 bullet lead-in (the bolded phrase before "—"): `text-body font-medium text-text`
- Step 1 bullet remainder: `text-body text-text-secondary`
- Step 2 + Step 3 heading: `text-heading text-text` (Inter, no serif — keep the Fraunces moment exclusive to Step 1)
- Step 2 + Step 3 body: `text-body text-text-secondary`
- Step indicator text: `text-label text-text-muted` ("Step 1 of 3" — locked over "1/3" / "Step 1 · 3" alternatives for screen-reader clarity)
- Skip-tour link: `text-label text-text-muted hover:text-text underline-offset-4 hover:underline`
- Replay button label: `text-label font-medium` (matches "Export Data" pattern at SettingsView.tsx:567)

## Color

60/30/10 split honored. The amber-light / violet-dark accent is the only color outside the neutral surfaces. No destructive color is used in this phase (no destructive actions).

| Role | Light Value | Dark Value | Usage |
|------|------|------|-------|
| Dominant (60%) | `var(--color-bg)` `#FFFFFF` | `var(--color-bg)` `#0A0A0D` | AlertDialog content background, Popover background, page beneath spotlight |
| Secondary (30%) | `var(--color-surface)` `#F9F8F6` / `var(--color-border)` `#E5E3DF` | `var(--color-surface)` `#141418` / `var(--color-border)` `#24242E` | Card borders, dot-indicator inactive state, Skip-tour link rest color, Help section icon-tile background (`bg-accent/10`) |
| Accent (10%) | `var(--color-accent)` `#D97706` (amber) | `var(--color-accent)` `#8B7FFF` (violet) | Active step-dot, privacy-bullet `<Check />` icons, primary CTA buttons (Continue / Try the FAB / Write your first entry) |
| Destructive | not used | not used | Phase 9 has zero destructive actions — Replay tour is non-destructive (just shows the tour again), so no destructive token referenced |

**Accent reserved for** (explicit list, never "all interactive elements"):

1. The active step-indicator dot (`bg-accent`) — communicates current progress
2. The 3 privacy-bullet `<Check />` icons in Step 1 (`text-accent`) — anchors the privacy promise
3. The primary CTA button on every step (`bg-accent text-amber-950` light / `bg-accent text-bg` dark — matches existing `OptionButton` selected state at SettingsView.tsx:29)
4. The HelpSection icon-tile background (`bg-accent/10`) and inner icon (`text-accent`) — matches DataSection, AppearanceSection, SecuritySection grammar

Inactive elements (do NOT receive accent):

- Inactive step-indicator dots → `bg-text-muted/40`
- Secondary CTA buttons (Skip tour link, "I'll explore first") → `text-text-muted` rest, `text-text` hover
- Body copy → `text-text-secondary` for descriptive prose, `text-text` for emphasis
- Spotlight backdrop → `rgba(0, 0, 0, 0.6)` (NOT a brand color — neutral darkening)


**z-index stack** (locked here from CONTEXT.md "Specific Ideas" coordination point):

| Layer | Class | Notes |
|-------|-------|-------|
| FAB at rest | `z-40` (existing) | unchanged from Phase 8 |
| FAB during onboarding Step 2 | `z-[80]` (raised) | Must remain interactive INSIDE the cutout — user can click it to advance |
| Spotlight backdrop | `z-[60]` | Below the FAB, below the popover |
| Spotlight cutout div | `z-[70]` (effectively, via `pointer-events-none`) | Visual only — the box-shadow IS the dim effect |
| Step 2 Popover content | `z-[70]` | Above backdrop, beside FAB |
| AlertDialog overlay (Steps 1 + 3) | `z-50` (shadcn default) | Phase 9 does NOT override — Steps 1 + 3 don't need to coexist with FAB |
| AlertDialog content (Steps 1 + 3) | `z-50` (shadcn default) | unchanged |

**ResizeObserver behavior:** Spotlight target rect recomputes on `window resize` AND on a `ResizeObserver` watching the FAB's bounding box. Both listeners are attached in `useLayoutEffect` and cleaned up on unmount (D-09 explicit). Scroll lock (`document.body.style.overflow = "hidden"`) applied during Step 2 only; restored on advance / skip.

### HelpSection (Settings) — locked structure (D-17)

```tsx
function HelpSection() {
  const [isReplaying, setIsReplaying] = useState(false);

  const handleReplay = async () => {
    setIsReplaying(true);
    try {
      await replayOnboarding();
    } finally {
      setIsReplaying(false);
    }
  };

  return (
    <section>
      <SectionHeader icon={<HelpCircle size={16} />} title="Help" />
      <div className="border-t border-border">
        <SettingRow
          label="Replay onboarding tour"
          description="Restart the welcome flow from the beginning"
        >
          <button
            onClick={handleReplay}
            disabled={isReplaying}
            className="flex items-center gap-1.5 px-3 py-1.5 text-label rounded-md border border-border text-muted hover:border-accent/50 hover:text-text transition-colors font-medium disabled:opacity-50"
          >
            {isReplaying ? (
              <>
                <Loader2 size={14} className="animate-spin" />
                Resetting...
              </>
            ) : (
              "Replay"
            )}
          </button>
        </SettingRow>
      </div>
    </section>
  );
}
```

**Insertion point:** Between `<DataSection />` and the version footer block in `SettingsView.tsx:608-611`. Insert order top-to-bottom remains: Appearance → Security → AI Features → Data → **Help (NEW)** → version footer.

**Pattern parity:** Section grammar (icon-tile + uppercase title + `border-t border-border` divider + SettingRow rows) is byte-identical to DataSection (`SettingsView.tsx:557-585`). Action button styling (`flex items-center gap-1.5 px-3 py-1.5 text-label rounded-md border border-border text-muted hover:border-accent/50 hover:text-text`) is byte-identical to "Export Data" button (`SettingsView.tsx:564-580`) MINUS the `<ChevronRight />` icon (per D-17, no chevron — replay is an inline action, not a nav).

## Accessibility Contract

| Surface | a11y guarantee |
|---------|----------------|
| AlertDialog (Steps 1 + 3) | Native Radix focus trap; Escape key triggers skip; Title and Description tied via `aria-labelledby` / `aria-describedby` automatically |
| Spotlight backdrop | `aria-hidden="true"` on the cutout div; the visible Popover IS the announced surface for screen readers |
| Step 2 Popover | Radix manages `role="dialog"` semantics; first focus lands on the primary CTA "Got it" |
| Skip-tour link | Native `<button>` with text `"Skip tour"` — screen readers announce intent clearly (D-15: NO icon-only X dismiss) |
| Step indicator | Decorative dots are `aria-hidden`; the text `"Step 1 of 3"` carries the semantic |
| HelpSection Replay button | Native `<button>` with text label; loading state preserves button width to prevent SR re-announcement of "now appearing" content |
| Spotlight target loss | Defensive: backdrop renders even if `data-onboarding="quick-write-fab"` is missing; user can still escape via Skip tour or Esc |
| Color contrast | Accent on background passes WCAG AA in both themes (light: amber #D97706 on #FFFFFF = 4.6:1; dark: violet #8B7FFF on #0A0A0D = 8.2:1) |
| Focus rings | Inherits global `focus-visible:ring-2 focus-visible:ring-primary` from button base; no custom focus styling needed |
| Keyboard navigation | Tab cycles within dialog (Radix focus trap); Enter on primary CTA advances; Escape closes (= skip) |
