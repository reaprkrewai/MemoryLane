# Phase 1 — UI Design Contract

> Visual and interaction contract for the Foundation phase. Establishes the global design system that all subsequent phases inherit. Generated by gsd-ui-researcher, verified by gsd-ui-checker.
>
> **Phase 1 UI scope:** Tauri window shell, app skeleton layout (sidebar + main content area), and the first-launch empty state. No entry editing, no timeline — those ship in Phases 2 and 3. Every design decision here becomes the baseline for all future phases.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | shadcn/ui (to be initialized in Phase 1 scaffold) |
| Preset | none — manual configuration via `npx shadcn@latest init` with Tailwind v3 |
| Component library | Radix UI primitives (via shadcn) |
| Icon library | lucide-react (shadcn default) |
| Font | Inter (system-stack fallback: ui-sans-serif, system-ui, sans-serif) |

**Source:** STACK.md — "shadcn/ui + Tailwind CSS v3", "Icons: lucide-react (shadcn default)"

**Tailwind version:** v3 only. shadcn/ui compatibility with Tailwind v4 was incomplete as of mid-2025. Do NOT upgrade to Tailwind v4 until shadcn officially supports it.

## Typography

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Body | 14px | 400 | 1.5 | Sidebar labels, metadata text, descriptions |
| Label | 12px | 400 | 1.4 | Tag pills, timestamps, word count indicators, secondary metadata |
| Heading | 20px | 600 | 1.2 | Panel titles, section headings |
| Display | 28px | 600 | 1.15 | Empty state headings, onboarding copy |

Label elements are differentiated from Body by size alone (12px vs 14px), both at weight 400. This keeps the weight system to exactly 2 values: 400 (regular) and 600 (semibold).

**Font family:** `Inter, ui-sans-serif, system-ui, sans-serif`

**Source:** CLAUDE.md design principles ("timeless design, focus on clarity and speed"), BRANDING.md ("clean, modern" typography direction). Sizes chosen to match a desktop-native journaling app — not a web SaaS — where 14px body at 1.5 line-height is comfortable at laptop viewing distances.

## Color

All values defined as Tailwind CSS custom properties in `tailwind.config.js` and as CSS variables on `:root` / `.dark`. Dark mode is a first-class requirement (SETT-01 in Phase 5, but the token system must support it from day 1).

| Role | Light Value | Dark Value | Usage |
|------|------------|------------|-------|
| Dominant (60%) | `#FAFAF9` (warm white) | `#111110` (near-black warm) | Window background, main content surface |
| Secondary (30%) | `#F0EFED` (warm off-white) | `#1C1B1A` (dark warm gray) | Sidebar background, nav rail, panel dividers |
| Accent (10%) | `#F59E0B` (warm amber) | `#F59E0B` (unchanged) | See reserved list below |
| Text Primary | `#1C1B1A` | `#FAFAF9` | All readable prose |
| Text Muted | `#6B7280` | `#9CA3AF` | Timestamps, secondary metadata, placeholder text |
| Border | `#E5E3DF` | `#2A2927` | Dividers, input borders, card outlines |
| Destructive | `#DC2626` | `#EF4444` | Destructive action buttons and error alerts only |

**Source:** BRANDING.md — "Primary: Deep blue/navy (#1e3a5f) — trust, privacy. Accent: Warm amber (#f59e0b) — memory, nostalgia. Background: Warm white (#fafaf9) — clarity."

**Note on navy:** The brand navy (#1e3a5f) is the logo/wordmark color and marketing surface color. For the app UI, the dominant surface is warm white #FAFAF9 (light) with near-black #111110 (dark) — navy is reserved for the app icon and onboarding illustration accents only. This avoids a corporate SaaS feel that conflicts with the "timeless, handcrafted" design goal.

**Accent reserved for:**
- The primary CTA button (single button per view, never more than one)
- Active sidebar navigation item indicator (left border or background pill)
- Mood indicator "great" state (the most positive mood color)
- Focus ring on the journal entry editor (Phase 2)
- Link text in empty state copy

Accent is NOT used for: hover states, generic interactive elements, tag colors, borders, or decorative elements.

## Empty State Visual Hierarchy

**Primary focal point:** Empty state display heading — "Your journal is ready" — rendered at 28px / weight 600 / line-height 1.15. This is the first element the eye lands on.

**Secondary:** A muted lucide icon at 48px, centered above the heading. Color: `text-muted` (`#6B7280` light / `#9CA3AF` dark). The icon does not compete with the heading — it reinforces it.

**Tertiary:** Body copy at 14px / weight 400 / line-height 1.5 directly below the heading.

**Quaternary:** The accent-colored CTA link ("Write your first entry") below the body copy.

Vertical stacking order (top to bottom): icon → heading → body → CTA. No card chrome, no border, no shadow. The empty state is a centered flex column with 16px gap between elements.

## Interaction Contracts

### Phase 1-specific interactions

| Interaction | Behavior |
|-------------|----------|
| App launch | Show loading spinner centered in main area while DB initializes (target: <300ms, imperceptible) |
| DB init success | Fade loading spinner out (150ms), fade empty state in (150ms) |
| DB init failure | Replace loading with error state (heading + body + "Restart app" label, no CTA button — user must restart manually) |
| Sidebar nav click | Immediate route change; no animation in Phase 1 (routes are stubs) |
| Window drag | Custom title bar region is draggable via `data-tauri-drag-region`; no content in that region |

### Transition spec (for Phase 2 executor reference)

All future view transitions use a single pattern: 150ms opacity fade. No slides, no scale — consistent with the "timeless, not trendy" design principle.

## Dark Mode Token Strategy

Tailwind CSS custom properties on `:root` and `.dark` class on `<html>`. The `uiStore` (Zustand) reads the saved theme from SQLite settings on launch and applies the class before first render to prevent flash.

```css
:root {
  --color-bg: #FAFAF9;
  --color-surface: #F0EFED;
  --color-accent: #F59E0B;
  --color-text: #1C1B1A;
  --color-text-muted: #6B7280;
  --color-border: #E5E3DF;
  --color-destructive: #DC2626;
}

.dark {
  --color-bg: #111110;
  --color-surface: #1C1B1A;
  --color-accent: #F59E0B;
  --color-text: #FAFAF9;
  --color-text-muted: #9CA3AF;
  --color-border: #2A2927;
  --color-destructive: #EF4444;
}
```

Map these to Tailwind `theme.extend.colors` in `tailwind.config.js` so that `bg-bg`, `bg-surface`, `text-accent`, etc. are available as utility classes.

**Source:** CLAUDE.md "Dark mode that's actually beautiful (not just inverted colors)", SETT-01 light/dark toggle.

## Design Quality Notes for Executor

These notes exist because CLAUDE.md explicitly warns against generic shadcn defaults:

1. **Do not use shadcn default blue accent.** Override `--primary` in `globals.css` to amber `#F59E0B` immediately after `npx shadcn init`.
2. **Sidebar is not a generic `Sheet` or `Drawer`.** It is a fixed-width column in the layout grid — not a slide-in panel.
3. **Empty state must not use a shadcn `Card`.** It is a centered flex column: icon (lucide, 48px, muted color) + display heading + body text + CTA link. No border, no shadow, no card chrome.
4. **Loading spinner:** Use `lucide-react`'s `Loader2` with `animate-spin` — no third-party spinner library.
5. **Typography:** Apply `font-feature-settings: "ss01"` on Inter for disambiguation of `1`, `l`, `I` — important for a journaling app where dates and numbers are prominent.
