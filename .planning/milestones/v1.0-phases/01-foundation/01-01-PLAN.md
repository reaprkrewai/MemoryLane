  <action>
    **Step 1 — Verify Rust is available:**
    Run `rustc --version` and `cargo --version`. If either fails, STOP and create a checkpoint:human-action asking the user to install Rust from https://rustup.rs and restart their terminal. Do NOT proceed without Rust.

    **Step 2 — Scaffold Tauri app:**
    From the project root (`C:\Users\Jason\Dev\MemoryLane`), run:
    ```bash
    npm create tauri-app@latest memorylane -- --template react-ts
    ```
    Then move all contents from the `memorylane/` subdirectory up to the project root (so `src/`, `src-tauri/`, `package.json` etc. are at root level). Remove the empty `memorylane/` directory.

    **Step 3 — Install npm dependencies:**
    ```bash
    npm install tailwindcss@3 @tailwindcss/typography autoprefixer postcss
    npm install @tauri-apps/plugin-sql
    npm install lucide-react sonner zustand
    ```
    Verify: `package.json` must contain `tailwindcss` at `^3.x` (NOT v4), `@tauri-apps/plugin-sql`, `lucide-react`, `sonner`, `zustand`.

    **Step 4 — Add Rust SQL plugin:**
    From `src-tauri/` directory, run:
    ```bash
    cargo add tauri-plugin-sql --features sqlite
    ```
    Verify: `src-tauri/Cargo.toml` contains `tauri-plugin-sql` with `features = ["sqlite"]`.

    **Step 5 — Configure tauri.conf.json:**
    Update `src-tauri/tauri.conf.json` — set the following in `app.windows[0]`:
    ```json
    {
      "label": "main",
      "title": "Chronicle",
      "width": 1200,
      "height": 800,
      "minWidth": 960,
      "minHeight": 600,
      "decorations": false,
      "titleBarStyle": "Overlay"
    }
    ```
    Set `productName` to `"Chronicle"` and `identifier` to `"com.chronicle.journal"`.

    **Step 6 — Register SQL plugin in Rust main.rs:**
    Update `src-tauri/src/main.rs` to register the SQL plugin:
    ```rust
    fn main() {
        tauri::Builder::default()
            .plugin(tauri_plugin_sql::Builder::default().build())
            .run(tauri::generate_context!())
            .expect("error while running tauri application");
    }
    ```

    **Step 7 — Set capabilities for SQL plugin:**
    Create or update `src-tauri/capabilities/default.json`:
    ```json
    {
      "identifier": "default",
      "description": "Default capability",
      "windows": ["main"],
      "permissions": [
        "core:default",
        "sql:default",
        "sql:allow-execute",
        "sql:allow-select",
        "sql:allow-load"
      ]
    }
    ```

    **Step 8 — Audit index.html for network calls (SETT-04):**
    Open `index.html`. Remove any `<link>` tags pointing to external URLs (Google Fonts, CDNs). Remove any `<script>` tags loading external resources. The only allowed external reference is none — everything must be local.

    **Step 9 — Verify compilation:**
    Run `npm run tauri dev`. The app must compile (both Rust and Vite) and open a native window. Close the app after confirming the window appears.

    **Step 1 — Initialize shadcn/ui:**
    Run `npx shadcn@latest init`. When prompted:
    - Style: Default
    - Base color: Slate (will be overridden)
    - CSS variables: Yes
    - Tailwind config: `tailwind.config.js`
    - Components directory: `src/components/ui`
    - Utils: `src/lib/utils.ts`

    If the CLI creates a `tailwind.config.ts` instead of `.js`, rename it to `tailwind.config.js` and convert to CommonJS `module.exports`.

    **Step 2 — Create postcss.config.js** (if not already created by shadcn init):
    ```javascript
    module.exports = {
      plugins: {
        tailwindcss: {},
        autoprefixer: {},
      },
    };
    ```

    **Step 3 — Configure tailwind.config.js:**
    Replace/update to exactly this content:
    ```javascript
    /** @type {import('tailwindcss').Config} */
    module.exports = {
      darkMode: 'class',
      content: ['./index.html', './src/**/*.{ts,tsx}'],
      theme: {
        extend: {
          colors: {
            bg: 'var(--color-bg)',
            surface: 'var(--color-surface)',
            accent: 'var(--color-accent)',
            text: 'var(--color-text)',
            muted: 'var(--color-text-muted)',
            border: 'var(--color-border)',
            destructive: 'var(--color-destructive)',
          },
          fontSize: {
            label: ['12px', { lineHeight: '1.4' }],
            body: ['14px', { lineHeight: '1.5' }],
            heading: ['20px', { lineHeight: '1.2', fontWeight: '600' }],
            display: ['28px', { lineHeight: '1.15', fontWeight: '600' }],
          },
        },
      },
      plugins: [],
    };
    ```

    **Step 4 — Create src/styles/globals.css:**
    Create the file with this exact content (overwrite any shadcn-generated globals):
    ```css
    @tailwind base;
    @tailwind components;
    @tailwind utilities;

    :root {
      --color-bg: #FAFAF9;
      --color-surface: #F0EFED;
      --color-accent: #F59E0B;
      --color-text: #1C1B1A;
      --color-text-muted: #6B7280;
      --color-border: #E5E3DF;
      --color-destructive: #DC2626;

      /* Override shadcn default blue accent */
      --primary: 40 96% 50%;
      --primary-foreground: 30 3% 10%;
    }

    .dark {
      --color-bg: #111110;
      --color-surface: #1C1B1A;
      --color-accent: #F59E0B;
      --color-text: #FAFAF9;
      --color-text-muted: #9CA3AF;
      --color-border: #2A2927;
      --color-destructive: #EF4444;
    }

    body {
      font-family: Inter, ui-sans-serif, system-ui, sans-serif;
      font-feature-settings: "ss01";
      background-color: var(--color-bg);
      color: var(--color-text);
    }
    ```

    **IMPORTANT:** Preserve any shadcn-generated CSS variables (like `--background`, `--foreground`, `--card`, etc.) that `components.json` expects. Append the Chronicle-specific variables above AFTER the shadcn variables, and ensure `--primary` is overridden to amber.

    **Step 5 — Update src/main.tsx:**
    Ensure `main.tsx` imports the globals CSS:
    ```typescript
    import React from "react";
    import ReactDOM from "react-dom/client";
    import App from "./App";
    import "./styles/globals.css";

    ReactDOM.createRoot(document.getElementById("root") as HTMLElement).render(
      <React.StrictMode>
        <App />
      </React.StrictMode>,
    );
    ```

    Remove any import of default Vite CSS (e.g., `import './index.css'` or `import './App.css'`). Delete the old CSS files if they exist (`src/index.css`, `src/App.css`).

    **Step 6 — Verify Tailwind classes work:**
    Temporarily add a test div to `App.tsx` with `className="bg-bg text-text"` and confirm it renders with the correct warm white background (#FAFAF9) and dark text (#1C1B1A) when running `npm run tauri dev`.
