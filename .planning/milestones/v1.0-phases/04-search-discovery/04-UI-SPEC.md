## Color

All values from `src/styles/globals.css` Chronicle design tokens. Do not introduce new color values.

| Role | Token | Hex (light) | Usage |
|------|-------|-------------|-------|
| Dominant (60%) | `--color-bg` / `bg-bg` | #FAFAF9 | SearchView background, results scroll area |
| Secondary (30%) | `--color-surface` / `bg-surface` | #F0EFED | Filter bar background, result cards (resting state), On This Day banner background |
| Accent (10%) | `--color-accent` / `bg-accent` | #F59E0B | See reserved list below |
| Destructive | `--color-destructive` | #DC2626 | Not used in this phase |
| Border | `--color-border` / `border-border` | #E5E3DF | Search input border, filter section divider, result card borders |
| Text primary | `--color-text` / `text-text` | #1C1B1A | Search input text, result preview text, chip labels |
| Text muted | `--color-text-muted` / `text-muted` | #6B7280 | Filter section labels, result metadata (date, word count), clear-all link, unselected chip text |

**Accent reserved for:**
1. Active sidebar nav item — "Search" left border stripe (3px, consistent with Phase 3 pattern)
2. Search keyword highlight — `<mark>` element background: `bg-accent/20` (amber at 20% opacity over surface)
3. Selected filter chip (tag, mood, date range active) — `bg-accent/15 border-accent/50 text-text` to signal active state
4. On This Day banner left border — 3px `border-l-accent` stripe to visually distinguish it from the standard timeline feed
5. "New Entry" button — unchanged from Phase 3 (amber primary fill)
6. Mood indicator dot on result cards — "great" mood only (unchanged from Phase 3 `bg-accent`)

Accent is NOT used for: the search input focus ring (use `ring-ring` shadcn token), unselected chips, result card hover (uses `bg-bg/60 border-l-accent` same as Phase 3), general borders.

### Highlight Mark Styling

```
<mark> element inside search result cards:
  background: color-mix(in srgb, #F59E0B 20%, transparent)  → bg-accent/20
  color: inherit (var(--color-text))
  border-radius: 2px
  padding: 0 2px
```

This is the canonical highlight style. All FTS5 match injections use this class. Do not use yellow `#FFFF00` or any other highlight color.

## Component Contracts

### SearchView Layout

Full-page view replacing the timeline/calendar content area when "Search" is active in the sidebar. Internal layout:

```
┌─────────────────────────────────────────────────────────┐
│  Search                                [Clear all]       │  ← Header row (sticky top-0)
│  ┌───────────────────────────────────────────────────┐  │
│  │ 🔍  Search entries...                             │  │  ← Search input
│  └───────────────────────────────────────────────────┘  │
│  ─────────────────────────────────────────────────────  │  ← divider
│  Date   [Start date ▾]  →  [End date ▾]                │  │  ← Date range row
│  Tags   [tag chip] [tag chip] [tag chip] ...            │  │  ← Tag filter row
│  Mood   [😂 Great] [😊 Good] [😐 Okay] [😞 Bad] [😡 Awful] │  ← Mood filter row
│  ─────────────────────────────────────────────────────  │  ← divider
│  [Result card]                                          │  │
│  [Result card]                                          │  │  ← Scrollable results
│  [Result card]                                          │  │
└─────────────────────────────────────────────────────────┘
```

- Max-width: `max-w-[720px] mx-auto px-4` — matches Phase 3 timeline width contract
- Header row: `sticky top-0 z-10 bg-bg` — "Search" heading (text-heading) + "Clear all" link (text-label text-muted) right-aligned. "Clear all" is visible only when any filter or keyword is active.
- Filters: always visible (no collapsible panel per CONTEXT.md decision). Rendered as a `bg-surface` panel with `rounded-md border border-border` and 16px padding.
- Scrollable area: results list below the filter panel, inherits page scroll.

### Search Input

```
┌──────────────────────────────────────────────────────┐
│  [Search icon 16px]   Search entries...               │
└──────────────────────────────────────────────────────┘
```

- shadcn `Input` component with `Search` lucide icon (16px, `text-muted`) inside the left of the input
- Placeholder: `"Search entries..."`
- Height: 40px (`h-10`)
- Border: `border-border` at rest, `ring-1 ring-ring` on focus (shadcn default — do not override with accent)
- Debounce: 300ms after keystroke before triggering FTS5 query (source: CONTEXT.md)
- Font: 14px body
- Clear: When the input contains text, show an `X` (lucide `X`, 14px, `text-muted hover:text-text`) inside the right of the input to clear keyword only

### Filter Panel

Three filter rows stacked vertically inside a panel container. Each row is a flex row with an 8px gap.

**Filter section label:** `text-label text-muted` at 56px minimum-width, aligned to the left of each row. Creates a visual column alignment between "Date", "Tags", "Mood" labels.

**Date Range Filter:**
- Two `DatePicker` instances (reuse `src/components/DatePicker.tsx`) for start and end date
- Displayed inline: `[Start date ▾]  →  [End date ▾]`
- Arrow separator: `→` in `text-muted text-label`, 8px horizontal margins
- Start date must be ≤ end date; if violated, clear the end date silently (no error toast)
- When a date is selected, the `DatePicker` button shows the date in `"MMM d, yyyy"` format
- When cleared, shows placeholder `"Start date"` / `"End date"` in `text-muted`

**Tag Filter:**
- Load all tags from the tag store; render as a wrapping flex row with 8px gap
- Each tag: a toggle chip using the tag's color (same `color-mix` pattern as `TagPillReadOnly`)
  - Unselected: `TagPillReadOnly` appearance — faint colored background, colored border
  - Selected: `background: color-mix(in srgb, {tag.color} 25%, transparent)`, `border-color: {tag.color}`, `font-weight: 600` — visually stronger but same color family
- Click toggles selected/unselected (multi-select)
- If no tags exist: show `"No tags yet"` in `text-label text-muted` in the tag row

**Mood Filter:**
- 5 mood pills, one per mood value, in order: Great → Good → Okay → Bad → Awful
- Each pill: `[emoji] [label]` in a `rounded-full` button shape
  - Emoji source: same MOODS constant from `MoodSelector.tsx` (import or replicate the 5-item array)
  - Unselected: `bg-surface text-muted border border-border rounded-full px-3 py-1 text-label`
  - Selected: `bg-accent/15 border-accent/50 text-text font-semibold rounded-full px-3 py-1 text-label`
- Click toggles selected/unselected (multi-select — user can filter "great + good" simultaneously)
- Minimum touch target: 32px height

### Search Result Card

Reuse `TimelineCard` component with an optional `highlights` prop. When `highlights` is provided, the card replaces the plain-text preview with a highlight-injected version.

```typescript
// TimelineCard receives:
interface TimelineCardProps {
  entry: TimelineCardEntry;
  tags: TimelineCardTag[];
  expanded: boolean;
  onToggleExpand: () => void;
  onOpen: () => void;
  highlights?: string; // HTML string with <mark> tags injected
}
```

- When `highlights` is present: render `<p dangerouslySetInnerHTML={{ __html: highlights }} />` replacing the plain-text preview. Sanitize to only allow `<mark>` and text nodes before injection.
- `<mark>` class: `bg-accent/20 rounded-[2px] px-[2px]`
- Card structure, spacing, hover, and expand-in-place behavior: unchanged from Phase 3 contract
- Result ordering: date descending (most recent first)
- All results shown (no pagination — user narrows with filters per CONTEXT.md)

### On This Day Banner

Appears at the top of the **TimelineView** scroll body (not in SearchView), above the first DaySeparator. Only rendered when past entries exist for today's calendar date in prior years.

```
┌──────────────────────────────────────────────────────┐
│ ▐ On This Day                          [▾ / ▴]       │
│   [entry card — 2025]                                 │
│   [entry card — 2024]                                 │
└──────────────────────────────────────────────────────┘
```

- Container: `rounded-md border border-border bg-surface` with 3px amber left border (`border-l-[3px] border-l-accent`)
- Padding: 12px vertical, 16px horizontal (`py-3 px-4`) — uses `xs-plus` (12px) token for vertical rhythm consistent with DaySeparator
- Header row: "On This Day" in `text-heading` (20px semibold) at left; chevron toggle button (`ChevronDown` / `ChevronUp`, 16px, `text-muted`) at right. Button `aria-label`: `"Collapse On This Day"` / `"Expand On This Day"`.
- Collapsed state: only the header row is visible (banner collapses to single row)
- Expanded state (default): entry cards shown below the header row, each in a compact read-only card variant (see below)
- Collapsed state persisted in `uiStore` (source: CONTEXT.md — "persist in uiStore")
- Year badge: Each OTD entry card shows a `text-label text-muted` year label (e.g., "2025", "2024") in place of the full date in the card header row

**OTD Entry Card (compact variant):**
- Same structure as `TimelineCard` but the date slot shows only the year: e.g., `"2025"` (`text-label text-muted`)
- No expand-in-place in OTD cards — clicking opens the entry directly in the editor
- No "word count" displayed in OTD cards (unnecessary at this density)
- Maximum 3 tags shown, same +N overflow as TimelineCard

### "Clear All" Control

Appears in the SearchView header row, right-aligned. Only visible when any filter is active (keyword, date range, any tag, any mood).

```
Clear all
```

- Plain text button (`text-label text-muted underline underline-offset-2 hover:text-text`)
- No icon
- Single click: resets keyword to empty, date range to null/null, tags to empty array, moods to empty array
- After clearing: SearchView shows unfiltered state (no results, only the search input and filters rendered)

### Empty / No-Results State

Shown in the SearchView results area when filters are active but no entries match.

```
┌─────────────────────────────────────────────────────────┐
│              [Search icon — 32px text-muted]            │
│              No entries found                           │
│         Try different keywords or fewer filters         │
└─────────────────────────────────────────────────────────┘
```

- Container: centered vertically in the available scroll area, `text-center`
- Icon: `Search` lucide (32px, `text-muted`)
- Heading: "No entries found" — `text-display` (28px semibold)
- Body: "Try different keywords or fewer filters." — `text-body text-muted` (14px)
- No CTA button in the no-results state

**Pre-search state** (SearchView open, nothing entered yet, no filters active):

```
              [Search icon — 32px text-muted]
              Search your journal
       Type a keyword or use the filters above
```

- Same layout as no-results state
- Heading: "Search your journal"
- Body: "Type a keyword or use the filters above."
