## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, tag gap, inline meta spacing |
| sm | 8px | Card inner element spacing, meta row gaps |
| md | 16px | Card horizontal padding, section padding |
| lg | 24px | Card vertical padding, timeline header padding |
| xl | 32px | Between day separator groups |
| 2xl | 48px | Timeline top margin, calendar top margin |
| 3xl | 64px | Page-level vertical spacing (not used in this phase) |

Exceptions:
- Day separator: 12px top/bottom margin (between sm and md — a deliberate half-step for reading rhythm)
- Calendar cell: minimum 40px height to ensure tap-friendly touch target on cells
- "Back" affordance in editor MetadataBar: 8px left padding, does not require full 16px

## Color

All values from `src/styles/globals.css` Chronicle design tokens. Do not introduce new color values.

| Role | Token | Hex (light) | Usage |
|------|-------|-------------|-------|
| Dominant (60%) | `--color-bg` / `bg-bg` | #FAFAF9 | App background, timeline scroll area, main content area |
| Secondary (30%) | `--color-surface` / `bg-surface` | #F0EFED | Sidebar, entry cards (default resting state), day separator background pill |
| Accent (10%) | `--color-accent` / `text-accent` / `bg-accent` | #F59E0B | See reserved list below |
| Destructive | `--color-destructive` / `text-destructive` | #DC2626 | Delete entry action only (no destructive actions in this phase) |
| Border | `--color-border` / `border-border` | #E5E3DF | Card borders, day separator rule, calendar cell borders |
| Text primary | `--color-text` / `text-text` | #1C1B1A | Card preview text, date labels, expanded entry prose |
| Text muted | `--color-text-muted` / `text-muted` | #6B7280 | Word count, tag pills in card, inactive calendar days |

**Accent reserved for:**
1. Active sidebar nav item — left border accent stripe (3px, already established)
2. Calendar heatmap: intensity level 2 (1 entry = `bg-accent/10`), level 3 (2–3 entries = `bg-accent/30`), level 4 (4+ entries = `bg-accent/60`)
3. "New Entry" button — primary action button fill
4. Entry card hover left border — a 2px amber left border appears on hover (signals clickability without card lift)
5. Mood indicator for "great" mood — amber color dot (the only mood that uses accent; others use muted)

Accent is NOT used for: general hover backgrounds, tag pills (those use their own 8-color system), body text, borders at rest.

### Heatmap Intensity Levels

| Level | Entries | Tailwind Class | Description |
|-------|---------|----------------|-------------|
| 0 | 0 entries | `bg-surface border border-border` | Faint surface, no amber |
| 1 | 1 entry | `bg-amber-100/40` (light) / `bg-amber-900/30` (dark) | Whisper of amber |
| 2 | 2–3 entries | `bg-accent/30` | Noticeable amber |
| 3 | 4+ entries | `bg-accent/60` | Strong amber — active writing day |

Use `--color-accent` with Tailwind opacity modifier syntax (`bg-accent/30`). Confirmed compatible with Tailwind v3.

### Mood Indicator Color Map (for timeline cards)

Mood indicators appear as a 8px filled circle on each card. Use these specific colors, consistent with the spirit of the existing `MoodIcon` pattern:

| Mood | Color | Tailwind |
|------|-------|---------|
| great | Amber | `bg-accent` |
| good | Emerald | `bg-emerald-400` |
| okay | Stone | `bg-stone-400` |
| bad | Orange | `bg-orange-400` |
| awful | Rose | `bg-rose-400` |
| none | (hidden) | — |

Use an 8px circle (`w-2 h-2 rounded-full`) as the mood indicator on timeline cards. No icon — the dot is faster to scan than an icon at card density.

## Component Contracts

### Timeline Entry Card

Structure (left to right, top to bottom):

```
┌─────────────────────────────────────────────────────┐
│ [mood dot]  [date string]              [word count]  │
│ [preview text — 150 chars plain text, ellipsis]      │
│ [tag pill] [tag pill] ...         [Expand ▾ / ▴]    │
└─────────────────────────────────────────────────────┘
```

- Background: `bg-surface` at rest, no shadow
- Border: `border border-border rounded-md` — 1px border, 8px radius
- Hover: amber left border `border-l-2 border-l-accent` + `bg-bg/60` — no card lift, no shadow
- Padding: 16px horizontal, 12px vertical (`px-4 py-3`)
- Cursor: `cursor-pointer`
- Click target: entire card navigates to editor (TIME-07)
- Expand trigger: chevron button `ChevronDown` (14px) aligned right, below the tag row. Clicking expands inline. When expanded, shows `ChevronUp`. The trigger label is visually icon-only but carries `aria-label="Expand entry"` / `aria-label="Collapse entry"`.
- Expanded state: full Markdown rendered as read-only prose below the preview text, within the same card bounds. Use same prose styles as `tiptap-editor` (font-size 14px, line-height 1.5). A "Collapse" text link appears below the prose (`text-muted text-label hover:text-text`).
- Tag pills in card: same `TagPill` component from Phase 2. Maximum 3 visible; if more, show `+N` muted label.
- Date string format: `Apr 9, 2026` — month abbreviated, no weekday on the card (weekday lives on the day separator).
- Word count format: `142w` — no "words" label.

### Day Separator

```
Wednesday, Apr 9  ─────────────────────────────────
```

- Full-width row between entries from different calendar days
- Left: date in `text-label font-semibold text-muted` format `"Wednesday, Apr 9"`
- Right: `<hr>` extending to the right edge, `border-border` color, 1px
- Vertical margin: 12px top and bottom (`my-3`)
- Not clickable. Not a filter trigger (that is Phase 4).

### Timeline Header Row

```
Journal                              [+ New Entry]
```

- Full-width row pinned to the top of the timeline scroll area (sticky, `sticky top-0 z-10 bg-bg`)
- Left: "Journal" in `text-heading` (20px, semibold)
- Right: "New Entry" button — shadcn `Button` variant `default` (amber primary fill via `--primary` token), size `sm`, label "+ New Entry"
- **Primary visual focal point:** The amber "+ New Entry" button is the primary visual anchor of the timeline header. The eye lands on it first due to the amber fill against the neutral bg-bg background.
- Padding: 16px horizontal, 16px vertical (`px-4 py-4`)
- Border bottom: `border-b border-border`

### Loading Sentinel / Spinner

- A centered `Loader2` (lucide-react, 20px) with `animate-spin` class
- Color: `text-muted`
- Appears below the last card when `hasMore === true` and the IntersectionObserver fires
- Disappears when next page loads
- No text label. No skeleton cards.

### Calendar View

Layout when "Calendar" nav is active — full main content area.

**Calendar Header:**
```
[< ChevronLeft]  April 2026  [ChevronRight >]  [Today]
```
- Centered month/year in `text-heading` (20px, semibold)
- `ChevronLeft` and `ChevronRight` lucide icons (20px), `text-muted hover:text-text`, 8px padding (`p-2`) for click target
- "Today" button: shadcn `Button` variant `ghost` size `sm`, appears at right of header row
- Padding: 24px all sides (`p-6`)

**Calendar Grid:**
- 7-column CSS grid (`grid grid-cols-7`)
- Row 1: day-of-week labels (`Sun Mon Tue Wed Thu Fri Sat`) — `text-label text-muted` centered in each column
- Remaining rows: calendar day cells
- Cell minimum height: 40px
- Cell padding: 4px
- Day number: `text-label` top-left within cell
- Today's date: circle indicator behind the day number — `bg-accent/20 rounded-full` on the number itself
- Days from previous/next month: `text-muted/40` (very faint), not clickable
- Cell hover (current month, has entries): `bg-surface cursor-pointer`
- Cell click: navigates to Journal view filtered to that date (CAL-04)

**Heatmap Cell Rendering:**
Apply intensity class to the cell background based on entry count for that day. Rounded: `rounded-md`. No border on colored cells (intensity > 0 cells drop the border; empty cells keep `border border-border`).

### "Back" Affordance in Editor

When the editor was opened from the timeline (not from the sidebar compact list), show a back button in the MetadataBar left zone:

```
[← ArrowLeft icon]  Back to Journal
```

- `ArrowLeft` lucide icon (14px) + label "Back to Journal"
- Color: `text-muted`, hover `text-text`
- Font: `text-label` (12px)
- Click: returns to Journal timeline view, scroll position preserved if possible
- Placed left of the existing MetadataBar content
