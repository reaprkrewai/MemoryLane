Create a new file `src/lib/ollamaService.ts` that implements a clean HTTP client for Ollama APIs:

**Exports:**
1. `checkOllamaHealth(): Promise<{ available: boolean; embedding: boolean; llm: boolean; }>`
   - GET to http://localhost:11434/api/tags
   - Timeout: 3 seconds
   - Returns { available: false, embedding: false, llm: false } on any error
   - Parses response to detect if "nomic-embed-text" and any "llama2" model are available
   - Per 06-RESEARCH.md Pattern 1

2. `generateEmbedding(content: string): Promise<number[]>`
   - POST to http://localhost:11434/api/embed
   - Request body: { model: "nomic-embed-text", input: content }
   - Returns 768-element float array (NOT 1536 per CLAUDE.md — actual model output)
   - Throws error if Ollama unavailable or request fails
   - Per 06-RESEARCH.md Pattern 2

3. `queryEmbedding(query: string): Promise<number[]>`
   - Same as generateEmbedding but for query text (reuses same endpoint)
   - Returns normalized query vector for vector similarity (Plan 06-02 will handle normalization)

4. `askQuestion(question: string, context: string): Promise<{ answer: string; citations: string[] }>`
   - POST to http://localhost:11434/api/generate
   - Request body: { model: "llama2:7b", system: systemPrompt, prompt: userPrompt, stream: false, temperature: 0.7, top_p: 0.9 }
   - systemPrompt: "You are a helpful assistant answering questions about journal entries. Always ground your answers in the provided entry content. When you reference information from an entry, cite it using [Entry ID]. If you cannot answer from the entries, say so explicitly."
   - userPrompt: "Based on these journal entries:\n\n{context}\n\nQuestion: {question}\n\nProvide a grounded answer with citations."
   - Parses response.response for [Entry XXXXX] pattern, extracts entry IDs
   - Returns { answer: rawAnswer, citations: [entryId1, entryId2, ...] }
   - Throws error if LLM model unavailable or request fails
   - Per 06-RESEARCH.md Pattern 4

**Error Handling:**
- All fetch() calls wrapped in try-catch
- Network errors, timeouts, malformed responses → throw Error with descriptive message
- Do NOT crash app; let calling code decide on graceful degradation

**No dependencies:** Plain fetch() (built-in to Tauri WebView)
