3. In the existing filtered tag row `div`, add the trash icon as the last child (after the usage count `span`). The icon must only render when `tag.usage_count === 0`. It must use `onMouseDown` (not `onClick`) to avoid the blur-race condition that applies to all elements in this dropdown:

   ```tsx
   {tag.usage_count === 0 && (
     <button
       type="button"
       className="ml-1 shrink-0 opacity-40 hover:opacity-100 hover:text-destructive transition-opacity"
       aria-label={`Delete tag ${tag.name}`}
       onMouseDown={(e) => {
         e.preventDefault();
         e.stopPropagation();
         deleteTag(tag.id);
       }}
     >
       <Trash2 size={13} />
     </button>
   )}
   ```

   Place this button AFTER the `<span className="text-label text-muted">{tag.usage_count}</span>` and BEFORE the closing `</div>` of the row.

**Why `onMouseDown` + `e.preventDefault()`:** The existing dropdown rows all use `onMouseDown` with `e.preventDefault()` to prevent the input's blur handler from firing before the click is processed (150ms blur timeout pattern established in Plan 03). The trash button must follow the same pattern. `e.stopPropagation()` prevents the row's own `onMouseDown` (which calls `onSelect`) from also firing.

**Why `opacity-40 hover:opacity-100`:** The trash icon should be subtle on zero-usage rows to avoid visual clutter, but clearly actionable on hover. This matches the existing hover-reveal pattern used for the entry delete icon in EntryListItem (`group-hover:opacity-100`).

**Do not** change the row's own `onMouseDown` handler. **Do not** add a new `onDelete` prop to TagAutocompleteProps — call `useTagStore` directly inside the component (same pattern as TagRow which reads `useTagStore` directly).

After these changes the component should be ~110 lines. TypeScript must compile with no errors.
